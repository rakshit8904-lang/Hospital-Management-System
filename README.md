# 🏥 Hospital Management System

## 📌 Project Overview

This project is a **Hospital Management System** 

The system efficiently manages:

* Patient registration
* Emergency prioritization
* Doctor assignment
* Bed allocation
* Patient records

---

## 🎯 Objectives

* Manage patient queues efficiently
* Prioritize emergency cases
* Assign doctors based on specialization
* Maintain patient records dynamically
* Simulate real-time hospital operations

---

## ⚙️ Technologies Used

* Python
* Django
* HTML/CSS
* Data Structures (Queue, Priority Queue, Linked List, Array)

---

## 🧠 Data Structures Used

### 🔹 Queue

* Used to manage **regular patient waiting list**
* First Come First Serve (FCFS)

### 🔹 Priority Queue

* Used for **emergency patients**
* Emergency patients are treated first

### 🔹 Linked List

* Stores **patient records dynamically**
* Efficient for large data handling

### 🔹 Array

* Stores:

  * Doctors
  * Bed availability
  * Departments

---

## 🔍 Algorithms Used

### 🔹 Searching

* Used to **find doctor based on disease/specialization**

### 🔹 Sorting

* Used to **arrange patients by priority or arrival time**

---

## 🧱 OOP Concepts Used

### Classes

* Patient
* Doctor
* Department
* Appointment
* HospitalManager

### Encapsulation

* Protects patient and doctor data

### Inheritance

* EmergencyPatient
* RegularPatient

### Polymorphism

* Different treatment methods for different patient types

---

## 🔄 System Workflow

1. Patient Registration
2. Queue Assignment (Regular / Emergency)
3. Doctor Allocation
4. Treatment Process
5. Bed Allocation
6. Record Storage

---

## 💻 Features

* Real-time patient queue system
